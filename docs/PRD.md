小学生作业管理APP - 产品技术方案

一、项目背景

痛点

1. 作业来源分散：语文老师在钉钉布置作业，英语补习班在微信群发内容（图片/视频），群消息量大，翻找困难，容易漏作业
2. 固定任务无提醒：每天练字等固定任务靠记忆管理，经常忘记
3. 多处打卡疲劳：各个平台各种打卡，操作繁琐

目标

做一个iPhone APP，打开就能看到今天孩子还有哪些作业没完成。支持从微信/钉钉群自动采集作业信息，也支持手动添加和定时重复任务。

用户

先自用跑通流程，iPhone用户，全栈开发者。

二、整体架构

架构决策

• 客户端：iOS原生（Swift + SwiftUI），最低 iOS 17

• 后端：NVP阶段不需要后端，所有数据本地存储 + iCloud自动同步

• 网络调用：仅在AI解析时直接从客户端调Claude API

• 架构模式：MVVM

技术栈

层 选择 理由

UI框架 SwiftUI 声明式UI，开发效率高

数据存储 SwiftData + iCloud 代码简洁，自动云同步，免后端

本地OCR Apple Vision框架 本地执行、免费、中文准确率95%+

AI解析 Claude API (Sonnet) 中文理解强，结构化输出稳定

录屏采集 ReplayKit Broadcast Upload Extension 实现跨APP内容捕获

通知 UNUserNotificationCenter 本地通知，不需要推送服务器

安全存储 iOS Keychain API Key安全存储

为什么不需要后端

SwiftData的iCloudKit同步可以免费实现多设备数据同步。AI解析直接客户端调API，等到需要钉钉机器人（服务端webhook）或多用户使用时再考虑后端。

三、核心功能设计

3.1 录屏自动采集（核心差异化功能）

这是APP的核心体验，采用本地OCR + 云端LLM混合方案。

用户操作流程

1. 首次设置：在APP中创建监控规则（见3.2节）
2. 开启权限：系统弹窗 → 用户确认 → 系统录屏接收确认（iOS安全机制，每次需确认一次，状态栏显示红色录屏指示）
3. 切到微信/钉钉 → 打开目标群聊
4. 自动采集：后台自动识别内容，含命中规则时手机顶部弹通知横幅“采集到 张老师 在 语文 群的 消息”
5. 查看结果：APP展示AI解析出的结构化作业列表，用户确认/编辑后保存

技术实现：两阶段过滤架构

第一级：本地Vision OCR + 群名快速过滤（低成本、低延迟）

• Broadcast Upload Extension每2秒采样一帧屏幕画面

• 与上一帧做像素级比较，无明显变化则跳过（省资源）

• 用Apple Vision框架对这个小区域做本地OCR识别群名

• 将识别出的群名与预设的监控规则群名列表匹配

• 匹配 → 进入第二级处理

• 不匹配 → 丢弃

为什么安全：只处理200x60px的小图片，Vision引擎内存开销极小（<10MB），远低于Extension的50MB限制。

第二级：云端Claude Vision API - 内容智能解析（高精度）

• 群名匹配成功后，将完整帧压缩为JPEG（~100KB）发送给Claude Vision API

• 发送结构化JSON提示词（Prompt），包含：确认群名、识别发言人、提取消息内容、判断是否包含作业、作业解析为结构化字段（科目、内容、截止日期、布置人）

• 命中监控规则中的“关注人”（如张老师）→ 发本地通知横幅 + 保存帧和解析结果到共享目录

• 不匹配 → 丢弃

为什么用云端：绕开Extension的50MB内存限制，不需要在Extension中加载完整的OCR引擎处理全屏内容。Claude Vision可以直接理解图片中的聊天界面结构，相比OCR+文字解析更准确。

已知限制

• 每次采集需系统弹窗确认录屏权限（iOS安全机制，无法跳过）

• 录屏时状态栏显示红色录屏指示（系统强制）

• 如果微信未来对录屏做屏幕的能力需要实验验证，如不可行则降级为录屏结束后统一通知

3.2 监控规则管理

规则创建方式：截图 + 一句话描述

• 用户在微信/钉钉里随便截一张群聊截图

• 上传到APP，贴入截图

• 输入文字：“关注张老师发的作业”

• APP用Vision OCR提取截图中的群聊文字，再调Claude API智能拆解为结构化规则

• 自动提取：群名称、关注的发言人、来源平台（微信/钉钉）、关键词（可选，如“作业”、“背诵”）

• 用户确认保存，也可手动编辑补充关键词等信息

规则数据结构

• 规则名称（用户自定义，如“语文作业”）

• 群名称（用于第一级本地OCR匹配）

• 来源平台（微信 / 钉钉）

• 关注发言人（如“张老师”）

• 关键词（可选，如“作业”、“背诵”）

• 是否启用

规则匹配逻辑

• 第一级：群名模糊匹配（容忍OCR识别的小误差）

• 第二级：发言人精确匹配 + 可选关键词匹配

• 匹配结果关联到对应的科目、自动归类

3.3 手动导入（补充采集方式）

截图导入：从相册选择截图 → Vision OCR提取文字 → Claude API解析 → 确认保存

文字粘贴导入：从微信/钉钉复制文字 → 粘贴到APP → Claude API解析 → 确认保存。打开APP时自动检测剪贴板是否有新内容。

3.4 手动添加任务

标准表单页面

• 科目选择（下拉选择，支持自定义添加新科目）

• 任务内容（多行文本）

• 截止日期（默认今天）

• 备注（可选）

• 附件（可选，拍照或选图）

3.5 重复任务

解决“每天练字”等固定任务容易忘记的问题。

• 创建重复规则：科目、内容、频率（每天/工作日/每周/自定义选择星期几）、每日提醒时间

• APP每次启动或进入前台时，自动检查所有活跃规则，为今天未完成的任务按规则生成待办

• 规则可暂停/恢复/删除

3.6 今日待办（主界面）

APP打开的默认页面，一眼看清今天还有什么没做。

• 按科目分组展示当日所有任务（语文、数学、英语、每日固定任务等）

• 每项任务：截止日期、完成按钮、记录完成时间

• 左滑删除，点击编辑

• 支持切换日期查看历史

• 底部操作：导入（触发录屏采集或手动导入）、手动添加

3.7 任务提醒

使用iOS本地通知。

• 有截止日期的任务：截止日当天早上提醒，当天下午仍未完成再提醒一次

• 每日固定任务（如“每日练字”）：每天固定时间触发，每日重复

• 提醒时间可在设置中统一配置

四、AI解析方案

模型选择

Claude Sonnet - 中文理解能力强，支持图片输入（Vision），结构化JSON输出稳定，价格合理。

两个使用场景

场景1：录屏帧解析（图片输入）

• 输入：微信/钉钉聊天界面的录屏截图（JPEG图片）

• 处理：调用Claude Vision API

• 输出：结构化JSON内容（群名称、发言人、消息内容，判断是否包含作业、作业解析为结构化字段（科目、内容、截止日期、布置人）

场景2：文字解析（截图OCR后的纯文本）

• 输入：OCR提取的聊天文字或用户粘贴文字

• 要求：从杂乱的聊天记录中过滤无关信息（家长闲聊、接龙、通知等），只提取作业相关内容

• 输出：结构化JSON（任务列表、布时间转换为具体日期）、布置人

调用策略

• Temperature=0，保证输出稳定

• System Prompt预设角色：作业信息提取助手，首次使用引导用户输入

• 相同文本内容做SHA256去重，避免重复调用

• 离线时暂存待处理内容，联网后自动补充解析

• 解析结果经用户确认后才保存到数据库

五、数据模型

作业任务（HomeworkTask）

核心实体，所有功能都围绕它展开。字段包括：第一ID、科目、任务内容、补充说明、截止日期、是否需完成、完成时间、来源类型（录屏采集/截图OCR/手动/重复任务自动生成）、来源详情（如“张老师”）、关联的重复规则ID。

重复规则（RecurringRule）

驱动固定任务的自动生成。字段包括：第一ID、科目、任务内容、重复频率（每天/工作日/每周/自定义）、自定义的星期几列表、每日提醒时间、是否启用、上次生成任务的日期、已生成的任务列表。

监控规则（MonitorRule）

录屏采集的配置。字段包括：第一ID、规则名称、群名称（用于本地OCR匹配）、关注的发言人列表、来源平台（微信/钉钉）、关键词列表（可选）、关联的科目、是否启用、创建时间。

附件（TaskAttachment）

关联到任务的图片或视频。字段包括：第一ID、类型（图片/视频）、本地文件路径、OCR提取的文字（如有）、创建时间。

科目（Subject）

可自定义的科目列表。字段包括：第一ID、名称、显示图标/emoji、排序顺序、是否默认、默认前缀：语文、数学、英语、科学。

导入记录（ImportRecord）

用于溯源和去重。字段包括：第一ID、原始文本的SHA256哈希（去重用）、原始文本内容、AI解析返回的JSON、来源类型、关联生成的任务列表、创建时间。

七、钉钉群的额外方案

通用方案（同微信）

录屏采集，与微信群的处理逻辑完全一致。钉钉界面的群名和发言人格式不同，AI解析时会自动适配。

增强方案（需管理员权限）

如果能获得学校钉钉的管理员或子管理员权限，可以创建企业内部应用+机器人：

• 机器人接收群内作业消息，通过Webhook推送到APP后端（需自建轻量后端，Node.js部署在CloudFlare Workers或Vercel免费额度）

• 需要将群设置为“允许应用发消息”，APP自动创建任务

• 此方案为后期演进路线，本期暂不实现

路径能力

• 微信：可通过URL Scheme weixin:// 打开微信APP，但无法打开指定群聊或定位到某条消息（微信未开放此能力）

• 钉钉：可通过URL Scheme dingtalk:// 打开钉钉APP，有部分deep link支持但无消息定位能力

• APP中记录来源群名和采集时间，辅助用户手动归类

八、开发路线图

Phase 0：技术验证（1-2天）

• 验证录屏采集扩展的可行性

• 创建Broadcast Upload Extension原型

• 验证采样帧 → 区域裁剪 → Vision OCR群名识别的内存消耗是否在50MB以内

• 验证Extension中能否成功调用Claude Vision API

• 此阶段决定方案是否可行，如不可行则降为纯手动导入方案

Phase 1：MVP核心功能（3个周末）

周末1：基础框架 + 数据层

• 创建Xcode项目，配置SwiftData和iCloud同步

• 实现数据模型定义

• 实现主界面框架（底部Tab导航）

• 实现手动添加任务和今日待办列表

周末2：采集 + AI解析

• 实现Broadcast Upload Extension + 帧采样 + 本地OCR + 云API + 通知

• 实现截图导入和粘贴导入

• 实现规则设置和Keychain存储

周末3：重复任务 + 提醒 + 打磨

• 实现重复规则的CRUD和每日任务自动生成

• 实现本地通知提醒

• 实现剪贴板新内容检测

• 整体磨合、UI打磨、边界情况处理

Phase 2：Share Extension（1个周末）

让用户从系统相册直接分享截图到APP，减少操作步骤。Extension中做OCR，结果写入共享目录，主APP打开后读取并调AI解析。

Phase 3：桌面小组件（1个周末）

WidgetKit小组件显示今日未完成任务数量和列表，支持Small和Medium两种尺寸，每小时自动刷新。

Phase 4：钉钉机器人集成（2个周末，需管理员权限）

搭建轻量后端，钉钉Stream模式对接，APNs推送集成。

Phase 5：历史统计（1个周末）

按周/月统计完成率、各科作业量分布、完成时间趋势，使用Swift Charts绘图。

6. 去重：向一条作业不会被重复录入
